🏗 Tool Architecture Overview

This tool is a single-page web application built with vanilla HTML, CSS, and JavaScript — deployable without a server. It is designed as a functional prototype for the IRO empirical study.

🖥 UI Layer

  • Responsive multi-section form with tab-based navigation
  • Likert/rating scales (1–5) for perception and readiness variables
  • Dropdown selects for categorical variables (ward, role, stream, method)
  • Auto-calculated variance field (actual vs target revenue)
  • Multi-checkbox anomaly pattern indicators
  • Client-side validation with real-time error highlighting
  • Live record table and ward-level summary aggregation

⚙️ Backend / Data Storage

Prototype Browser localStorage (JSON serialization)

Production Recommended: PostgreSQL or MongoDB with a Node.js/FastAPI backend. Records map cleanly to a relational schema (respondents → revenue_records → system_ratings).

  • Data types collected: Continuous (KES amounts), ordinal (Likert scales), nominal (ward, role, stream), binary (anomaly flags)
  • Export formats: CSV (for SPSS/R/Excel), JSON (for Python/ML pipelines)
  • Records persist across browser sessions via localStorage

🔒 Ethics & Validation

  • Consent Informed consent gate before data entry
  • Privacy No names collected — respondent IDs only
  • Validation Required fields enforced; numeric ranges checked
  • Anonymity Ward-level aggregation masks individual responses
  • Data Minimisation Only variables necessary for IRO research collected

📐 Variables & Rationale

Variables were selected based on the three research objectives:

  • Revenue performance data (target, actual, variance) → supports Objective 1: profiling collection patterns
  • Transaction counts & anomaly flags → supports Objective 3: anomaly detection model training
  • Likert scales on ML readiness & system reliability → supports Objective 4: integration feasibility assessment
  • Historical data availability & format → supports Objective 2: forecasting model feasibility
